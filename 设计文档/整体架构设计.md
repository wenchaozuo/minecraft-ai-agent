## 一、项目愿景

**目标**：构建一个能够自主游玩 Minecraft、最终通关击杀末影龙的 AI 智能体。

**核心能力**：
- 接受自然语言指令
- 自主规划目标
- 持续自主决策
- 从失败中学习
- **通过视觉理解环境和建筑**

## 四、数据流

```
1. JS层采集感知（文本）
   ↓ getFullPerception()
   {environment, threat, resources, progress}

2. 【新增】JS层视觉渲染（按需）
   ↓ captureVision()
   screenshot PNG (base64)

3. POST /agent/tick + POST /vision
   ↓ Python接收感知 + 截图

4. 【新增】VLM 分析截图
   ↓ VisionAnalyzer.analyze()
   {structures, terrain, threats}

5. 合并文本感知 + 视觉分析
   ↓ merge_perception()
   {text, vision, summary}

6. 记忆系统更新
   ↓ memory.update_world_state()

7. 规划器获取当前目标
   ↓ planner.get_current_goal()

8. 构建 LLM 上下文（含视觉数据）
   ↓ memory.get_context_for_llm()

9. ReAct 决策
   ↓ reactor.decide(context)
   {think, action, expected}

10. 执行动作
    ↓ bot_client.execute_action()
    {success: true, count: 3}

11. 记录到记忆
    ↓ memory.add_action()

12. 反思（如果失败）
    ↓ critic.reflect()

13. 循环
```

## 五、API接口

### 现有接口(保留)

| 接口 | 说明 |
|------|------|
| `/chat` | 玩家聊天指令 |
| `/action` | 简单避障 |

### 新增接口

| 接口 | 方向 | 说明 |
|------|------|------|
| `/agent/start` | - | 启动自主决策循环 |
| `/agent/stop` | - | 停止自主决策循环 |
| `/agent/status` | - | 获取Agent状态 |
| `/agent/tick` | JS → Python | 自主模式单次决策 |
| `/perception` | JS → Python | 获取当前感知 |
| **`/vision`** | **JS → Python** | **发送截图，接收VLM分析结果** |
| **`/vision/request`** | **Python → JS** | **请求特定视角的截图** |

## 七、技术选型

| 组件 | 技术 | 说明 |
